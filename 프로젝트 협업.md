1. Create : 회원가입, 평점 리뷰(아이디당 한번), 좋아요 등록, 
2. Read : 스와이프, 음식점 상세페이지

### 데이터베이스 설계

#### 1. 요구사항 분석

        1. 푸드득을 이용하기 위해선 회원가입을 해야한다.
           - 아이디
           - 비밀번호
           - 닉네임
           - 개인이메일
     2. 회원은 회원아이디로 식별한다.
     3. 음식점에 대한 테이블을 만든다.
        - 음식점 코드
        - 이름
        - 종류
        - 위치
        - 이미지경로
     4. 음식점은 음식점 코드로 식별한다.
     5. 회원은 '좋아요' 한 음식점을 따로 관리한다. 
        - 좋아요 id
        - 회원 아이디(f)
        - 음식점 코드(f)
        - 날짜
     6. 회원은 '싫어요' 하면, 후순위로 한번 더 보여주고 또 '싫어요' 누를 시 그날은 그 음식점에 대해서 나오지 않는다. (2~3일마다 리셋)
     7. 회원마다 '싫어요' 테이블은 만들어야 할까?
        - 싫어요 id
        - 회원 아이디(f)
        - 음식점 코드(f)
        - 날짜
     8. 회원은 마이페이지에서 각 종 설정값을 바꿀 수 있다. (테이블이 있어야 하나?)
